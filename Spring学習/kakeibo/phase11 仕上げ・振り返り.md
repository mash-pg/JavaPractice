## 次のステップ

### 推奨: 部分練習

全部作り直すより、**苦手な部分だけを「見ないで書く」練習** が効率的。

| 練習対象 | 時間目安 | 例 |
|---------|---------|-----|
| Value Object | 5〜10分 | Amount, CategoryName を白紙から書く |
| UseCase | 10〜15分 | DeleteCategoryUseCase を白紙から書く |
| Controller → UseCase → Repository | 20〜30分 | 1機能だけ流れを書く |

**進め方:**
1. 「仕様」だけを見る（コードは見ない）
2. 書いてみる
3. 答え合わせ（実際のコードと比較）
4. 間違った部分を確認

### その後

```
→ 部分練習（1週間で5〜7回）
→ 予約システム（3個目）
→ 拡張（Docker、認証など）
→ DDD 深掘り
```
